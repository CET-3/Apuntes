Arquitectura de Internet y aplicaciones web

Este apunte explica cómo viaja la información por internet y cómo se organizan las aplicaciones web entre cliente, servidor, base de datos y APIs.

1. ¿Qué es internet?

Internet es una red enorme de computadoras y dispositivos conectados entre sí. Gracias a esa red, una máquina puede pedir información y otra puede responder.

Cliente

Es el dispositivo que hace el pedido, por ejemplo una computadora o un celular.

Servidor

Es la computadora que recibe el pedido y responde con información.

Red

Es el conjunto de conexiones que permite que los datos viajen de un lugar a otro.

2. IP, dominio y DNS

Para encontrar un sitio en internet hacen falta direcciones. Algunas son numéricas y otras son nombres más fáciles de recordar.

IP

Es una dirección numérica que identifica a una máquina en la red.

Dominio

Es el nombre que usamos para entrar a un sitio, por ejemplo google.com.

DNS

Es el sistema que traduce un dominio a una dirección IP.

Idea simple: el dominio es el nombre, la IP es la dirección y el DNS es quien hace la traducción.

3. ¿Qué pasa cuando escribimos una URL?

  1. La persona escribe una dirección en el navegador.
  2. El navegador busca la IP usando DNS.
  3. Se conecta con el servidor.
  4. El servidor responde con archivos o datos.
  5. El navegador muestra el contenido en pantalla.

4. HTTP y HTTPS

HTTP

Es el protocolo que se usa para enviar y recibir información web.

HTTPS

Es una versión más segura de HTTP porque protege la comunicación.

En la mayoría de los sitios actuales se usa https:// porque mejora la seguridad.

5. ¿Qué es una aplicación web?

Una aplicación web es un programa que usamos desde el navegador. Puede mostrar información, recibir datos del usuario y trabajar con lógica del lado del servidor.

Sitio web

Suele mostrar contenido, como textos, imágenes y páginas informativas.

Aplicación web

Además de mostrar contenido, permite interactuar, guardar datos y realizar acciones.

6. Frontend, backend y base de datos

Frontend

Es la parte que ve y usa la persona: botones, textos, formularios, imágenes.

Backend

Es la parte que procesa pedidos, aplica lógica y prepara respuestas.

Base de datos

Es donde se guarda la información del sistema.

Frontend → Backend → Base de datos
Frontend ← Backend ← Base de datos

7. ¿Qué es una API?

Una API es una forma de comunicación entre programas. En una aplicación web, el frontend puede pedir datos a una API y el backend responde con la información.

Pedido

El frontend solicita información, por ejemplo una lista de productos.

Respuesta

El backend devuelve los datos para que el frontend los muestre.

8. Ejemplo completo

Imaginemos una aplicación para ver notas escolares.

  1. La persona entra al sitio desde el navegador.
  2. El frontend muestra la pantalla inicial.
  3. El usuario inicia sesión.
  4. El frontend envía el pedido al backend.
  5. El backend consulta la base de datos.
  6. La base de datos devuelve la información.
  7. El backend responde al frontend.
  8. El frontend muestra las notas en pantalla.
