Introducción a VS Code, GitHub y Vercel

Esta página resume las ideas básicas que hace falta conocer para trabajar con un proyecto, escribir código, guardarlo en GitHub y publicarlo en internet.

1. Introducción a las tres herramientas

VS Code

Es el editor donde escribimos código y organizamos los archivos del proyecto.

GitHub

Es una página web donde guardamos el proyecto y su historial de cambios.

Vercel

Es una plataforma que publica el proyecto en internet para que otras personas puedan verlo.

Idea simple: VS Code sirve para crear, GitHub sirve para guardar y Vercel sirve para publicar.

2. Por qué usar un editor de código

Un editor de código está pensado para programar. No solo deja escribir, sino que también ayuda a entender y organizar mejor el trabajo.

Ayuda visual

Colorea etiquetas, muestra mejor el código y hace más fácil encontrar errores.

Más orden

Permite trabajar con carpetas, archivos y sangrías de una manera más clara.

Herramientas útiles

Puede autocompletar, marcar errores y conectarse con Git y GitHub.

Idea simple: un editor de texto sirve para escribir. Un editor de código sirve para escribir código mejor.

3. Por qué trabajar con GitHub

GitHub permite guardar el proyecto en internet y llevar un registro de los cambios. También facilita el trabajo en equipo.

Copia de seguridad

Si la computadora falla, el proyecto sigue guardado en GitHub.

Historial

Permite ver qué cambió, cuándo cambió y volver atrás si hace falta.

Trabajo compartido

Hace más fácil colaborar y acceder al proyecto desde distintas computadoras.

Idea simple: GitHub sirve para guardar, compartir y ordenar la historia del proyecto.

4. Qué hace Vercel

Vercel toma el proyecto y lo convierte en una página web accesible desde internet. Así, el trabajo deja de estar solo en la computadora o en GitHub y pasa a estar publicado.

Publicación

Genera una dirección web para abrir el proyecto desde cualquier navegador.

Actualización

Puede volver a publicar la página cuando hacemos cambios nuevos en el repositorio.

Compartir

Permite mostrar el proyecto a otras personas sin pasarles la carpeta completa.

Idea simple: GitHub guarda el proyecto y Vercel lo muestra en internet.
