Git y GitHub

Este apunte explica cómo guardar cambios, trabajar con un repositorio y usar GitHub para compartir el proyecto y publicarlo con Vercel.

HTML básico Introducción VS Code Git y GitHub

1. Palabras importantes

Git

Es la herramienta que registra los cambios del proyecto.

Repositorio

Es la carpeta del proyecto que está controlada con Git.

Commit

Es una foto guardada de los cambios que elegiste.

Push

Es la acción de subir tus commits a GitHub.

Pull

Es la acción de traer cambios nuevos desde GitHub a tu computadora.

Clone

Es bajar un proyecto por primera vez a tu computadora.

Branch

Es una rama de trabajo dentro del proyecto.

2. Changes y Staged Changes

En la pestaña Source Control de VS Code aparecen los archivos modificados.

Changes

Son cambios que hiciste, pero todavía no preparaste para guardar.

Staged Changes

Son cambios que ya seleccionaste para que entren en el próximo commit.

Comando equivalente:

# Agrega un archivo al área de preparación
git add index.html

También se puede agregar otro archivo puntual:

# Agrega un archivo específico
git add style.css

Idea simple: staged significa listo para guardar.

3. Cómo clonar un proyecto desde la UI

Si querés bajar un proyecto por primera vez usando la interfaz de VS Code, los pasos son estos:

  1. Abrir VS Code.
  2. Apretar Ctrl + Shift + P.
  3. Escribir Git: Clone.
  4. Pegar la dirección del repositorio.
  5. Elegir una carpeta para guardarlo.
  6. Apretar Open cuando termine.

Desde la UI, clonar significa usar la opción Git: Clone para bajar el proyecto sin escribir comandos.

Comando equivalente en terminal:

# Descarga el proyecto por primera vez
git clone https://github.com/CET-3/propuesta1.git

Clone significa bajar el proyecto por primera vez.

4. Cómo hacer un commit desde la UI

  1. Ir a Source Control.
  2. Buscar los archivos modificados.
  3. Tocar el signo + para pasarlos a Staged Changes.
  4. Escribir un mensaje corto y claro.
  5. Tocar Commit.

Comandos equivalentes:

# Prepara el archivo para el commit
git add index.html

# Guarda una foto de los cambios
git commit -m "Agrega página inicial"

Ejemplos de mensajes:

Agrega página inicial
Corrige colores
Separa estilos en CSS

5. Cómo hacer push y pull

Push

Después del commit, hay que tocar Push o Sync Changes para subirlo a GitHub.

Pull

Antes de empezar a trabajar, conviene hacer Pull para traer los cambios nuevos.

Comandos equivalentes:

# Sube los cambios a GitHub
git push

# Trae cambios nuevos desde GitHub
git pull

6. Relación con Vercel

Cuando GitHub está conectado con Vercel, cada vez que hacemos push, Vercel puede detectar los cambios y publicar una nueva versión del sitio.

GitHub

Guarda el proyecto y recibe los cambios que subimos.

Vercel

Publica el proyecto y puede actualizar la versión online automáticamente.

Idea simple: GitHub guarda el trabajo y Vercel lo publica en internet.

7. Orden recomendado de trabajo

  1. Abrir el proyecto.
  2. Hacer Pull si hace falta actualizar.
  3. Modificar los archivos.
  4. Revisar cambios en Source Control.
  5. Pasarlos a Staged Changes.
  6. Hacer Commit.
  7. Hacer Push.

Ejemplo completo por terminal:

# Actualiza la copia local
git pull

# Prepara un archivo para el commit
git add style.css

# Guarda los cambios con un mensaje
git commit -m "Actualiza el proyecto"

# Sube los cambios a GitHub
git push

8. Requisitos

Para usar GitHub y publicar con Vercel, primero hace falta crear una cuenta en cada plataforma.

Cuenta de GitHub

Creala desde la página oficial:

https://github.com/signup

Cuenta de Vercel

Creala desde la página oficial:

https://vercel.com/signup/

Requisitos básicos: tener una cuenta en GitHub para guardar el proyecto y una cuenta en Vercel para publicarlo.

Resumen rápido: clone baja, pull actualiza, commit guarda y push sube.
