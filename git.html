Git y GitHub

Este apunte explica cómo guardar cambios, trabajar con un repositorio y usar GitHub para compartir el proyecto y publicarlo con Vercel.

1. ¿Para qué usamos GitHub en este proyecto?

En esta materia no usamos GitHub porque "así trabajan los programadores" solamente. Lo usamos porque nos ayuda a ordenar, guardar, compartir y publicar el proyecto de la mutual escolar.

Guardar

El proyecto queda guardado en internet y no solo en una computadora.

Ordenar

Permite llevar una historia de cambios sin terminar con archivos llamados final, final2 o final-ahora-si.

Compartir

Hace más fácil que varias personas trabajen sobre el mismo proyecto.

Publicar

Al conectarlo con Vercel, lo que subimos puede terminar publicado en internet.

Idea simple: GitHub guarda la historia del proyecto y nos ayuda a llevarlo desde la computadora hasta la web publicada.

2. ¿Qué herramientas intervienen?

Para este recorrido vamos a usar varias herramientas distintas, y cada una cumple una función diferente.

VS Code

Es el lugar donde escribimos y modificamos los archivos del proyecto.

Git

Es la herramienta que registra los cambios del proyecto.

GitHub

Es la página donde guardamos el proyecto y su historial de cambios.

Vercel

Es la plataforma que toma el proyecto y lo publica en internet.

3. Palabras básicas para empezar

Repositorio

Es la carpeta del proyecto guardada con control de versiones.

Commit

Es una foto guardada del proyecto en un momento determinado.

Push

Es subir los commits al repositorio en GitHub.

Pull

Es traer cambios nuevos desde GitHub a la computadora.

Clone

Es bajar un proyecto por primera vez desde GitHub.

Deploy

Es publicar una nueva versión del proyecto en internet.

Frases para recordar: commit es una foto, push es subir, pull es traer y deploy es publicar.

4. ¿Qué problema resuelve GitHub?

Si no usamos una herramienta para ordenar el trabajo, suelen aparecer varios problemas.

Sin GitHub

Se pierden archivos, aparecen muchas versiones distintas y cuesta trabajar en grupo.

Con GitHub

El proyecto queda guardado, se puede compartir, se puede volver atrás y es más fácil colaborar.

5. Cómo crear un proyecto o repositorio nuevo

A veces no vamos a bajar un proyecto ya hecho, sino a empezar uno desde cero. En ese caso primero se crea la carpeta del proyecto y después se la conecta con GitHub.

  1. Crear una carpeta para el proyecto en la computadora.
  2. Abrir esa carpeta en VS Code.
  3. Entrar a Control de código fuente.
  4. Inicializar el repositorio para que Git empiece a registrar la historia del proyecto: hacer clic en Inicializar repositorio o, si no aparece ese botón, abrir la paleta con Ctrl+Shift+P y buscar Git: Inicializar repositorio.
  5. Publicar el proyecto en GitHub desde la UI de VS Code para que se cree el repositorio remoto y quede conectado con el proyecto local.

Como validacion, tienen que poder ver el proyecto en GitHub. Si aparece ahi, ya quedo listo para hacer commits y subir cambios, algo que vamos a ver despues.

Idea simple: primero existe la carpeta del proyecto, después Git empieza a guardar su historia y luego GitHub la recibe en internet.

6. Cómo clonar un proyecto desde la UI

Clonar es bajar el proyecto por primera vez desde GitHub a tu computadora.

  1. Abrir VS Code.
  2. Apretar Ctrl + Shift + P.
  3. Escribir Git: Clone.
  4. Pegar la dirección del repositorio.
  5. Elegir una carpeta para guardarlo.
  6. Apretar Open cuando termine.

Comando equivalente en terminal:

# Descarga el proyecto por primera vez
git clone https://github.com/CET-3/propuesta1.git

7. Cambios y cambios preparados

Para poder subir algo, primero hace falta crear o modificar archivos en el proyecto. En la pestaña Control de código fuente de VS Code aparecen esos cambios y suelen organizarse en dos grupos importantes.

Cambios

Son archivos que cambiaste, pero que todavía no marcaste para guardar en el próximo commit.

Cambios preparados

Son archivos que sí elegiste incluir en el próximo commit.

Cuando tocás el signo + al lado de un archivo, ese archivo pasa de Cambios a Cambios preparados.

Comando equivalente:

# Agrega un archivo al área de preparación
git add index.html

También se puede agregar otro archivo puntual:

# Agrega un archivo específico
git add style.css

Idea simple: primero hacés cambios, después elegís cuáles querés preparar y recién ahí hacés el commit.

8. Cómo guardar cambios

Cuando hacemos un cambio en el proyecto, normalmente el recorrido es: elegir archivos, escribir un mensaje y guardar esa nueva versión.

  1. Modificar un archivo.
  2. Revisar los cambios en Control de código fuente.
  3. Pasarlos a Cambios preparados.
  4. Escribir un mensaje claro.
  5. Hacer Commit para guardar esa foto.

Comandos equivalentes:

# Prepara el archivo para el commit
git add index.html

# Guarda una foto de los cambios
git commit -m "Agrega página inicial"

Ejemplos de mensajes:

Agrega página inicial
Corrige colores
Separa estilos en CSS

9. Cómo hacer push

Después de hacer cambios y guardarlos con commit, el paso siguiente es hacer push. Eso significa subir esa nueva versión del proyecto a GitHub.

  1. Hacemos un cambio en VS Code.
  2. Guardamos la nueva versión con commit.
  3. La subimos a GitHub con push.

Comando equivalente:

# Sube la nueva versión del proyecto a GitHub
git push origin main

Idea simple: commit guarda la versión en la historia del proyecto y push la sube a GitHub.

10. Qué pasa después del push

Cuando GitHub está conectado con Vercel, el proyecto no solo queda guardado: también puede terminar publicado en internet.

  1. Hacemos un cambio en VS Code.
  2. Guardamos la nueva versión con commit.
  3. La subimos a GitHub con push.
  4. Vercel detecta el cambio.
  5. Vercel publica la nueva versión del sitio.

Idea simple: cuando el proyecto ya está conectado, cada push puede terminar en una nueva versión publicada.

12. Requisitos

Para usar GitHub y publicar con Vercel, primero hace falta crear una cuenta en cada plataforma.

Cuenta de GitHub

Creala desde la página oficial:

https://github.com/signup

Cuenta de Vercel

Creala desde la página oficial:

https://vercel.com/signup/

Requisitos básicos: tener una cuenta en GitHub para guardar el proyecto y una cuenta en Vercel para publicarlo.
