Git y GitHub en VS Code

Esta página resume las ideas básicas que hace falta conocer para trabajar con un proyecto, guardar cambios y subirlos a GitHub usando la interfaz gráfica de VS Code.

HTML básico Git y GitHub

1. Qué es cada cosa

Git

Es una herramienta que sirve para guardar cambios en un proyecto.

GitHub

Es una página web donde los proyectos pueden guardarse y compartirse por internet.

VS Code

Es el editor donde escribimos código y también podemos usar Git con botones.

2. Por qué usar un editor de código

Un editor de código está pensado para programar. No solo deja escribir, sino que también ayuda a entender y organizar mejor el trabajo.

Ayuda visual

Colorea etiquetas, muestra mejor el código y hace más fácil encontrar errores.

Más orden

Permite trabajar con carpetas, archivos y sangrías de una manera más clara.

Herramientas útiles

Puede autocompletar, marcar errores y conectarse con Git y GitHub.

Idea simple: un editor de texto sirve para escribir. Un editor de código sirve para escribir código mejor.

3. Por qué trabajar con GitHub

GitHub permite guardar el proyecto en internet y llevar un registro de los cambios. También facilita el trabajo en equipo.

Copia de seguridad

Si la computadora falla, el proyecto sigue guardado en GitHub.

Historial

Permite ver qué cambió, cuándo cambió y volver atrás si hace falta.

Trabajo compartido

Hace más fácil colaborar y acceder al proyecto desde distintas computadoras.

Idea simple: GitHub sirve para guardar, compartir y ordenar la historia del proyecto.

4. Palabras importantes

Repositorio

Es la carpeta del proyecto que está controlada con Git.

Commit

Es una foto guardada de los cambios que elegiste.

Push

Es la acción de subir tus commits a GitHub.

Pull

Es la acción de traer cambios nuevos desde GitHub a tu computadora.

Clone

Es bajar un proyecto por primera vez a tu computadora.

Branch

Es una rama de trabajo dentro del proyecto.

5. Changes y Staged Changes

En la pestaña Source Control de VS Code aparecen los archivos modificados.

Changes

Son cambios que hiciste, pero todavía no preparaste para guardar.

Staged Changes

Son cambios que ya seleccionaste para que entren en el próximo commit.

Comando equivalente:

# Agrega un archivo al área de preparación
git add index.html

También se puede agregar otro archivo puntual:

# Agrega un archivo específico
git add style.css

Idea simple: staged significa listo para guardar.

6. Cómo clonar un proyecto desde la UI

  1. Abrir VS Code.
  2. Apretar Ctrl + Shift + P.
  3. Escribir Git: Clone.
  4. Pegar la dirección del repositorio.
  5. Elegir una carpeta para guardarlo.
  6. Apretar Open cuando termine.

Comando equivalente:

# Descarga el proyecto por primera vez
git clone https://github.com/CET-3/propuesta1.git

Clone significa bajar el proyecto por primera vez.

7. Cómo hacer un commit desde la UI

  1. Ir a Source Control.
  2. Buscar los archivos modificados.
  3. Tocar el signo + para pasarlos a Staged Changes.
  4. Escribir un mensaje corto y claro.
  5. Tocar Commit.

Comandos equivalentes:

# Prepara el archivo para el commit
git add index.html

# Guarda una foto de los cambios
git commit -m "Agrega página inicial"

Ejemplos de mensajes:

Agrega página inicial
Corrige colores
Separa estilos en CSS

8. Cómo hacer push y pull

Push

Después del commit, hay que tocar Push o Sync Changes para subirlo a GitHub.

Pull

Antes de empezar a trabajar, conviene hacer Pull para traer los cambios nuevos.

Comandos equivalentes:

# Sube los cambios a GitHub
git push

# Trae cambios nuevos desde GitHub
git pull

9. Orden recomendado de trabajo

  1. Abrir el proyecto.
  2. Hacer Pull si hace falta actualizar.
  3. Modificar los archivos.
  4. Revisar cambios en Source Control.
  5. Pasarlos a Staged Changes.
  6. Hacer Commit.
  7. Hacer Push.

Ejemplo completo por terminal:

# Actualiza la copia local
git pull

# Prepara un archivo para el commit
git add style.css

# Guarda los cambios con un mensaje
git commit -m "Actualiza el proyecto"

# Sube los cambios a GitHub
git push

10. Enlace a la otra página

Un enlace entre páginas del mismo proyecto se hace con la etiqueta <a>.

<a href="index.html">Volver a HTML básico</a>

Probá este enlace: Volver a la página de HTML básico

Resumen rápido: clone baja, pull actualiza, commit guarda y push sube.
