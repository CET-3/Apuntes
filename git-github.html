Git y GitHub en VS Code

Esta página resume las ideas básicas que hace falta conocer para trabajar con un proyecto, guardar cambios y subirlos a GitHub usando la interfaz gráfica de VS Code.

1. Qué es cada cosa

Git

Es una herramienta que sirve para guardar cambios en un proyecto.

GitHub

Es una página web donde los proyectos pueden guardarse y compartirse por internet.

VS Code

Es el editor donde escribimos código y también podemos usar Git con botones.

2. Palabras importantes

Repositorio

Es la carpeta del proyecto que está controlada con Git.

Commit

Es una foto guardada de los cambios que elegiste.

Push

Es la acción de subir tus commits a GitHub.

Pull

Es la acción de traer cambios nuevos desde GitHub a tu computadora.

Clone

Es bajar un proyecto por primera vez a tu computadora.

Branch

Es una rama de trabajo dentro del proyecto.

3. Changes y Staged Changes

En la pestaña Source Control de VS Code aparecen los archivos modificados.

Changes

Son cambios que hiciste, pero todavía no preparaste para guardar.

Staged Changes

Son cambios que ya seleccionaste para que entren en el próximo commit.

Idea simple: staged significa listo para guardar.

4. Cómo clonar un proyecto desde la UI

  1. Abrir VS Code.
  2. Apretar Ctrl + Shift + P.
  3. Escribir Git: Clone.
  4. Pegar la dirección del repositorio.
  5. Elegir una carpeta para guardarlo.
  6. Apretar Open cuando termine.

Clone significa bajar el proyecto por primera vez.

5. Cómo hacer un commit desde la UI

  1. Ir a Source Control.
  2. Buscar los archivos modificados.
  3. Tocar el signo + para pasarlos a Staged Changes.
  4. Escribir un mensaje corto y claro.
  5. Tocar Commit.
Ejemplos de mensajes:
Agrega página inicial
Corrige colores
Separa estilos en CSS

6. Como hacer push y pull

Push

Después del commit, hay que tocar Push o Sync Changes para subirlo a GitHub.

Pull

Antes de empezar a trabajar, conviene hacer Pull para traer los cambios nuevos.

7. Orden recomendado de trabajo

  1. Abrir el proyecto.
  2. Hacer Pull si hace falta actualizar.
  3. Modificar los archivos.
  4. Revisar cambios en Source Control.
  5. Pasarlos a Staged Changes.
  6. Hacer Commit.
  7. Hacer Push.

8. Enlace a la otra página

Un enlace entre páginas del mismo proyecto se hace con la etiqueta <a>.

<a href="index.html">Volver a HTML básico</a>

Probá este enlace: Volver a la página de HTML básico

Resumen rápido: clone baja, pull actualiza, commit guarda y push sube.
